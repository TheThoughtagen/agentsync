  <name>Task 1: Make ClaudeCode adapter use processed content when conditionals apply</name>
  <files>crates/aisync-core/src/adapters/claude_code.rs</files>
  <behavior>
    - Test 1: When canonical_content equals raw file content (no conditionals), plan_sync returns CreateSymlink (existing behavior preserved)
    - Test 2: When canonical_content differs from raw file content (conditionals applied), plan_sync returns CreateFile with the processed content instead of CreateSymlink
    - Test 3: When CLAUDE.md is already a regular file with correct processed content, plan_sync returns empty actions (idempotent)
    - Test 4: When CLAUDE.md is a symlink but conditionals now apply, plan_sync returns remove symlink + CreateFile
  </behavior>
  <action>
Modify `ClaudeCodeAdapter::plan_sync()` to actually USE the `canonical_content` parameter (currently named `_canonical_content`):

1. **Rename `_canonical_content` to `canonical_content`** — stop ignoring it.

2. **Read the raw canonical file** to compare: Read `.ai/instructions.md` directly to get the unprocessed content. If `canonical_content` (processed) equals the raw content, use the existing symlink logic (no conditionals active for this tool).

3. **When content differs (conditionals applied):** Instead of creating a symlink, return `SyncAction::CreateFile { path: link_path, content: canonical_content.to_string() }`. This writes the processed content as a regular file.

4. **Handle existing state transitions:**
   - No CLAUDE.md exists + no conditionals -> CreateSymlink (unchanged)
   - No CLAUDE.md exists + conditionals -> CreateFile with processed content
   - CLAUDE.md is symlink + no conditionals -> check target, return empty or RemoveAndRelink (unchanged)
   - CLAUDE.md is symlink + conditionals -> RemoveAndRelink is wrong; instead remove symlink and CreateFile
   - CLAUDE.md is regular file + conditionals -> compare content hash, return empty if matches or CreateFile if differs
   - CLAUDE.md is regular file + no conditionals -> this is the existing SkipExistingFile case (user-created file, not aisync-managed)

5. **Detecting aisync-managed regular files:** To distinguish between "user created CLAUDE.md" and "aisync wrote CLAUDE.md with processed conditionals", check if the file was previously a symlink OR if the content matches processed output. The simplest approach: when conditionals are active, always write the processed file (overwrite). When conditionals are NOT active, preserve existing SkipExistingFile behavior for regular files.

The key insight: `plan_sync` receives `canonical_content` which is ALREADY processed by `ConditionalProcessor` in `SyncEngine::plan()`. The adapter just needs to detect whether processing changed anything (compare against raw file) and choose symlink vs write accordingly.

To get the raw content for comparison, read `.ai/instructions.md` from `project_root`. This is a small additional read but necessary to determine strategy.
