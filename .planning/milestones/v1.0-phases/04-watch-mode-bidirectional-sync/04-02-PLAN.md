   ```rust
   /// Event emitted by the watch engine for logging/display.
   #[derive(Debug, Clone)]
   pub enum WatchEvent {
       ForwardSync { changed_path: PathBuf },
       ReverseSync { tool: ToolKind, source_path: PathBuf },
       Error { message: String },
   }
   ```

2. Create `crates/aisync-core/src/watch.rs`:
   - NOTE: This module uses `notify` and `notify-debouncer-mini` which are dependencies of the `aisync` CLI crate, not `aisync-core`. Since the watch engine needs access to SyncEngine (in core), design it so the core module provides the sync logic (what to do on events) while the actual file watching (notify setup) lives in the CLI crate.
   - Actually, re-evaluate: the watch engine can live in core if we add notify deps to core. Add `notify = { workspace = true }` and `notify-debouncer-mini = { workspace = true }` to `crates/aisync-core/Cargo.toml` instead.
   - Create `pub struct WatchEngine;` following the established pattern (struct with associated functions).
   - Implement `pub fn watch(config: &AisyncConfig, project_root: &Path, running: Arc<AtomicBool>, event_callback: impl Fn(WatchEvent)) -> Result<(), AisyncError>`:
     - Create an `Arc<AtomicBool>` for `syncing` (the sync lock)
     - Create an mpsc channel for debounced events
     - Initialize `notify_debouncer_mini::new_debouncer` with 500ms duration
     - Watch `.ai/` directory recursively via `debouncer.watcher().watch()`
     - Compute tool watch paths: for each enabled tool, determine the specific files to watch (CLAUDE.md, AGENTS.md, .cursor/rules/project.mdc). Only watch paths that exist and are NOT symlinks (symlinks don't need reverse sync).
     - Watch each tool-native file path with `NonRecursive` mode
     - Enter event loop: `for events in rx { ... }`
       - Check `running.load(SeqCst)` -- if false, break (graceful shutdown)
       - Check `syncing.load(SeqCst)` -- if true, continue (skip self-triggered events)
       - Classify events: `is_canonical` if any path starts with `.ai/`, `is_tool_native` otherwise
       - Set `syncing = true`
       - If `is_tool_native && !is_canonical`: call `Self::reverse_sync()`, then forward sync
       - If `is_canonical`: just forward sync via `SyncEngine::plan()` + `SyncEngine::execute()`
       - Call `event_callback` with appropriate WatchEvent
       - Set `syncing = false`
   - Implement `fn tool_watch_paths(config: &AisyncConfig, project_root: &Path) -> Vec<PathBuf>`:
     - For each enabled tool, return the tool-native instruction file path:
       - ClaudeCode: `project_root.join("CLAUDE.md")`
       - OpenCode: `project_root.join("AGENTS.md")`
       - Cursor: `project_root.join(".cursor/rules/project.mdc")`
     - Filter to only existing, non-symlink files (check with `symlink_metadata()`)
   - Implement `fn reverse_sync(config: &AisyncConfig, project_root: &Path, changed_paths: &[PathBuf]) -> Result<Option<WatchEvent>, AisyncError>`:
     - For each changed path, determine which tool it belongs to
     - Get the appropriate adapter and call `read_instructions(project_root)`
     - If content differs from current `.ai/instructions.md`, write the tool-native content to `.ai/instructions.md`
     - Return WatchEvent::ReverseSync if content was updated
     - For Cursor: adapter's read_instructions() already strips frontmatter, so the content is clean

3. Update `crates/aisync-core/src/lib.rs`:
