    - validate: checks event names against VALID_EVENTS (PreToolUse, PostToolUse, Notification, Stop, SubagentStop)
    - validate: returns HookError::InvalidEvent for unknown event names
    - list_hooks: returns flat list of (event, matcher, command, timeout) tuples

  </behavior>
  <action>
    Create `crates/aisync-core/src/hooks.rs` implementing `HookEngine`:

    ```rust
    pub const VALID_EVENTS: &[&str] = &[
        "PreToolUse", "PostToolUse", "Notification", "Stop", "SubagentStop",
    ];

    pub struct HookEngine;

    impl HookEngine {
        /// Parse .ai/hooks.toml into HooksConfig.
        pub fn parse(project_root: &Path) -> Result<HooksConfig, AisyncError>;

        /// Validate all event names in a HooksConfig against VALID_EVENTS.
        pub fn validate(config: &HooksConfig) -> Result<(), AisyncError>;

        /// Flatten hooks into a list of (event, matcher, command, timeout) for display.
        pub fn list_hooks(config: &HooksConfig) -> Vec<HookSummary>;

        /// Add a new hook to .ai/hooks.toml. Creates the file if it doesn't exist.
        /// Appends a new [[Event]] entry.
        pub fn add_hook(
            project_root: &Path,
            event: &str,
            matcher: Option<&str>,
            command: &str,
            timeout: Option<u64>,
        ) -> Result<(), AisyncError>;

        /// Serialize HooksConfig back to TOML string.
        pub fn serialize(config: &HooksConfig) -> Result<String, AisyncError>;
    }

    pub struct HookSummary {
        pub event: String,
        pub matcher: Option<String>,
        pub command: String,
        pub timeout: Option<u64>,
    }
    ```

    For `add_hook`: read existing file (or create empty HooksConfig), deserialize, add new entry to the appropriate event key, serialize back. Use `toml::to_string_pretty()`.

    For `parse`: `std::fs::read_to_string` then `toml::from_str::<HooksConfig>()`. The `#[serde(flatten)]` on the BTreeMap handles the `[[EventName]]` TOML syntax.

    Test TOML round-trip carefully -- the `flatten` + `BTreeMap<String, Vec<HookGroup>>` pattern with TOML arrays-of-tables needs explicit verification.

    Update `lib.rs` to add `pub mod hooks;` and export `HookEngine, HookSummary, VALID_EVENTS`.

    Override `translate_hooks` on `ClaudeCodeAdapter`. Build Claude Code JSON structure:
    ```json

    }

    Key details:

    - Omit `matcher` key entirely when it's None (not null, just absent)

          execSync('npm run lint', { timeout: 10000 });
        },
      },
    };
    ```
