<task type="auto" tdd="true">
  <name>Task 1: Create RuleEngine module and wire into sync pipeline</name>
  <files>crates/aisync-core/src/rules.rs, crates/aisync-core/src/lib.rs, crates/aisync-core/src/sync.rs</files>
  <behavior>
    - RuleEngine::load() reads .ai/rules/*.md files and returns Vec<RuleFile> sorted by name
    - RuleEngine::load() returns empty vec when .ai/rules/ directory doesn't exist
    - RuleEngine::load() returns empty vec when .ai/rules/ has no .md files
    - parse_frontmatter() extracts YAML frontmatter (description, globs, always_apply) between --- delimiters
    - parse_frontmatter() handles missing frontmatter (entire file as content, default metadata with always_apply=true)
    - parse_frontmatter() handles globs in both array [*.rs, *.toml] and comma-separated "*.rs, *.toml" format
    - parse_frontmatter() handles quoted and unquoted description values
    - SyncEngine::plan_all_internal() loads rules and passes them to each adapter's plan_rules_sync()
  </behavior>
  <action>
    **Create `crates/aisync-core/src/rules.rs`:**

    Implement `RuleEngine` struct with:
    - `pub fn load(project_root: &Path) -> Result<Vec<RuleFile>, AisyncError>` -- reads `.ai/rules/` directory, parses each `.md` file, returns sorted Vec<RuleFile>. Use `std::fs::read_dir`, filter to `.md` extension, sort by name for deterministic ordering.
    - `fn parse_frontmatter(raw: &str) -> Result<(RuleMetadata, String), AisyncError>` -- splits on `---\n` delimiters. The opening `---` must be the very first characters in the file. Find closing `---` using `\n---\n` or `\n---` at end of string (NOT just `find("---")` which would match inside YAML values). Extract YAML key-value pairs via hand-parsing (no YAML library).
    - `fn parse_yaml_metadata(yaml_str: &str) -> Result<RuleMetadata, AisyncError>` -- hand-parse `description:`, `globs:`, `always_apply:` lines. For globs: handle both `[*.rs, *.toml]` array syntax and `"*.rs, *.toml"` comma-separated string. Trim quotes from values. Default: `always_apply=true`, `description=None`, `globs=vec![]`.

    **Update `crates/aisync-core/src/lib.rs`:**
    - Add `pub mod rules;` declaration
    - Add `pub use rules::RuleEngine;` to the re-exports

    **Update `crates/aisync-core/src/sync.rs` -- `plan_all_internal()`:**
    - After the `memory_files` line, add: `let rules = crate::rules::RuleEngine::load(project_root)?;`
    - Inside the per-tool loop, after the hook translation block and before `results.push(...)`, add:
      ```rust
      if !rules.is_empty() {
          match adapter.plan_rules_sync(project_root, &rules) {
              Ok(rule_actions) => actions.extend(rule_actions),
              Err(e) => {
                  actions.push(SyncAction::WarnUnsupportedDimension {
                      tool: tool_kind.clone(),
                      dimension: "rules".into(),
                      reason: format!("rule sync failed: {e}"),
                  });
              }
          }
      }
      ```

    Write unit tests in `rules.rs` for all behaviors listed above. Use `tempfile::TempDir` for filesystem tests.
  </action>

<task type="auto" tdd="true">
  <name>Task 2: Implement CursorAdapter and WindsurfAdapter plan_rules_sync</name>
  <files>crates/aisync-core/src/adapters/cursor.rs, crates/aisync-core/src/adapters/windsurf.rs</files>
  <behavior>
    - CursorAdapter::plan_rules_sync() generates aisync-{name}.mdc files in .cursor/rules/ with Cursor frontmatter
    - Cursor frontmatter maps: description -> description, globs -> comma-separated quoted string, always_apply -> alwaysApply (camelCase)
    - CursorAdapter::plan_rules_sync() returns CreateDirectory if .cursor/rules/ doesn't exist
    - CursorAdapter::plan_rules_sync() scans for stale aisync-*.mdc files and returns RemoveFile actions
    - CursorAdapter::plan_rules_sync() returns empty vec for empty rules input
    - CursorAdapter::plan_rules_sync() skips file generation when content is unchanged (idempotent)
    - WindsurfAdapter::plan_rules_sync() generates aisync-{name}.md files in .windsurf/rules/ with Windsurf frontmatter
    - Windsurf frontmatter maps: always_apply=true -> trigger: always_on; globs present -> trigger: glob; description only -> trigger: model_decision; fallback -> trigger: manual
    - WindsurfAdapter::plan_rules_sync() scans for stale aisync-*.md files and returns RemoveFile actions
    - WindsurfAdapter::plan_rules_sync() returns empty vec for empty rules input
  </behavior>
  <action>
    **Update `crates/aisync-core/src/adapters/cursor.rs`:**

    Add helper functions:
    - `fn generate_cursor_rule_frontmatter(meta: &RuleMetadata) -> String` -- produces Cursor-format YAML frontmatter. Maps: `description` -> `description:`, `globs` -> `globs: "{joined}"` (comma-separated, quoted), `always_apply` -> `alwaysApply:` (camelCase boolean).
    - `fn generate_cursor_rule_content(meta: &RuleMetadata, body: &str) -> String` -- combines frontmatter + body.

    Implement `plan_rules_sync()` on CursorAdapter:
    1. If rules is empty, return empty vec.
    2. Build `expected` HashSet of filenames: `aisync-{rule.name}.mdc` for each rule.
    3. Ensure `.cursor/rules/` directory exists (CreateDirectory if needed).
    4. For each rule: generate Cursor-format content. Check if `aisync-{name}.mdc` already exists with same content (idempotent skip). Otherwise emit `CreateRuleFile { output, content, rule_name }`.
    5. Scan `.cursor/rules/` for existing `aisync-*.mdc` files NOT in expected set -> emit `RemoveFile` for each stale file.
    6. Return all actions.

    **Update `crates/aisync-core/src/adapters/windsurf.rs`:**

    Add helper functions:
    - `fn generate_windsurf_rule_frontmatter(meta: &RuleMetadata) -> String` -- produces Windsurf-format YAML frontmatter. Logic: if `always_apply` -> `trigger: always_on`; else if globs present -> `trigger: glob` + `globs: {joined}`; else if description present -> `trigger: model_decision`; else -> `trigger: manual`. Always include `description:` if present.
    - `fn generate_windsurf_rule_content(meta: &RuleMetadata, body: &str) -> String` -- combines frontmatter + body.

    Implement `plan_rules_sync()` on WindsurfAdapter:
    1. Same pattern as Cursor but with `.windsurf/rules/` directory and `.md` extension.
    2. Expected filenames: `aisync-{rule.name}.md`.
    3. Use `CreateRuleFile` for new/changed files, `RemoveFile` for stale files.

    **Important:** Use `use std::collections::HashSet;` for the expected filename set. Import `RuleFile` from `crate::types` (or `aisync_adapter::aisync_types`).

    Write unit tests for both adapters covering all behaviors. Use `tempfile::TempDir`. Test frontmatter generation separately from plan_rules_sync to keep tests focused.
