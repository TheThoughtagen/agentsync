  <action>
    **Pattern for all three adapters:**

    Each adapter implements `plan_rules_sync()` with the same logic, differing only in the target file path (`self.native_instruction_path()`). Create a shared helper function or implement inline on each:

    ```rust
    fn plan_rules_sync(
        &self,
        project_root: &Path,
        rules: &[RuleFile],
    ) -> Result<Vec<SyncAction>, AdapterError> {
        if rules.is_empty() {
            return Ok(vec![]);
        }
        let mut content = String::new();
        for rule in rules {
            if !rule.content.is_empty() {
                content.push_str(&format!("\n## Rule: {}\n\n", rule.name));
                content.push_str(&rule.content);
                content.push('\n');
            }
        }
        if content.is_empty() {
            return Ok(vec![]);
        }
        Ok(vec![SyncAction::UpdateMemoryReferences {
            path: project_root.join(self.native_instruction_path()),
            references: vec![content],
            marker_start: "<!-- aisync:rules -->".to_string(),
            marker_end: "<!-- /aisync:rules -->".to_string(),
        }])
    }
    ```

    **For ClaudeCodeAdapter** (`crates/aisync-core/src/adapters/claude_code.rs`):
    - Add `use aisync_adapter::aisync_types::RuleFile;` if not already imported
    - Override `plan_rules_sync()` with the pattern above, targeting `project_root.join("CLAUDE.md")`

    **For OpenCodeAdapter** (`crates/aisync-core/src/adapters/opencode.rs`):
    - Same pattern, targeting `project_root.join("AGENTS.md")`

    **For CodexAdapter** (`crates/aisync-core/src/adapters/codex.rs`):
    - Same pattern, targeting `project_root.join("AGENTS.md")`

    **Note on deduplication:** OpenCode and Codex both target AGENTS.md. The existing `deduplicate_actions()` in sync.rs handles path conflicts for `CreateSymlink`/`CreateFile`/`RemoveAndRelink` but NOT for `UpdateMemoryReferences`. Since both produce the same markers and same content for AGENTS.md, the managed_section updater is idempotent -- running it twice produces the same result. This is acceptable for v1.2.

    Write unit tests for each adapter. Use `tempfile::TempDir`. Create mock `RuleFile` instances with known content and verify the returned `SyncAction::UpdateMemoryReferences` has correct path, markers, and concatenated content.
