Create `docs/ADAPTER-AUTHORING.md` with the following structure:

1. **Introduction** - Brief overview of aisync's two-layer adapter system: TOML for simple adapters, Rust for complex ones.

2. **TOML Adapter Authoring** section covering:
   - Where to place TOML files (`.ai/adapters/mytool.toml`)
   - Complete TOML schema reference with all fields:
     - `name`, `display_name` (required)
     - `[detection]` with `directories`, `files`, `match_any` fields
     - `[sync]` with `instruction_path`, `strategy` fields
     - `[template]` with `content` field (optional, for Generate strategy)
   - Full working example: an Aider adapter TOML file
   - How detection works (match_any vs match_all)
   - Template syntax (`{{content}}` interpolation)
   - Limitations (no custom hooks, no memory sync, no custom status logic)

3. **Rust Adapter Authoring** section covering:
   - When to choose Rust over TOML (custom detection logic, hooks, memory sync, complex output generation)
   - Crate setup: `Cargo.toml` with `aisync-adapter` and `inventory` dependencies
   - Implementing `ToolAdapter` trait: required methods (`name`, `display_name`, `native_instruction_path`, `detect`) and optional methods with defaults
   - Registering with `inventory::submit!` using `AdapterFactory`
   - Linking into the binary: adding the crate to `aisync/Cargo.toml` (or user's binary)
   - Warning about `extern crate` if linker strips unused crate
   - Full working example referencing `examples/adapter-example/`

4. **How It Works** section:
   - Discovery order: builtins first, then TOML adapters, then inventory adapters
   - Name deduplication: first-seen wins (builtin > TOML > inventory)
   - How `enabled_tools()` and `aisync.toml` `[tools.X]` interact

5. **Troubleshooting** section:
   - "My adapter isn't detected" (check marker files, check detection config)
   - "My Rust adapter isn't picked up" (check it's in Cargo.toml deps, check linking)
   - "Name collision warning" (rename adapter, builtin names are reserved)

Keep the doc factual and reference-oriented. No marketing language. Use code blocks for all examples.
