**1. Create `crates/aisync/src/commands/add_tool.rs`:**

`pub fn run_add_tool(tool_flag: Option<&str>, verbose: bool) -> Result<(), Box<dyn std::error::Error>>`

Flow:
1. `let project_root = std::env::current_dir()?;`
2. Check `aisync.toml` exists (same pattern as sync.rs):
   ```rust
   let config_path = project_root.join("aisync.toml");
   if !config_path.exists() {
       eprintln!("{}", "No aisync.toml found. Run `aisync init` first.".red());
       std::process::exit(1);
   }
   ```
3. `let mut config = AisyncConfig::from_file(&config_path)?;`
4. Branch on `tool_flag`:

**Non-interactive path (`--tool <name>` provided):**
- Parse tool name to ToolKind via `ToolKind::from(name)` (the FromStr/From impl)
- Actually ToolKind doesn't have FromStr. Use a simple match or iterate `AnyAdapter::all_builtin()` matching on `as_str()`. If not found, error with "Unknown tool: {name}. Available: claude-code, cursor, opencode, windsurf, codex"
- Check if already configured: `if config.tools.get_tool(name).is_some()` → print "{tool} is already configured" and return Ok
- Call `AddToolEngine::add_tools(&mut config, &[tool_kind], &project_root)?`
- Run partial sync: `let report = SyncEngine::plan_for_tools(&config, &project_root, &[tool_kind])?;`
- Execute: `let executed = SyncEngine::execute(&report, &project_root)?;`
- Print summary (tool name + number of actions executed)

**Interactive path (no --tool flag):**
- `let interactive = std::io::stdin().is_terminal();`
- Call `AddToolEngine::discover_unconfigured(&config, &project_root)?`
- If empty: print "All detected tools are already configured." in green, return Ok
- If not interactive (piped stdin, CI):
  - List unconfigured tools
  - Print hint: "Use `aisync add-tool --tool <name>` in non-interactive mode."
  - Return Ok
- Use `dialoguer::MultiSelect` for interactive selection:
  ```rust
  use dialoguer::MultiSelect;
  let items: Vec<String> = unconfigured.iter()
      .map(|d| format!("{} ({:?})", d.tool.display_name(), d.confidence))
      .collect();
  let selections = MultiSelect::new()
      .with_prompt("Select tools to add")
      .items(&items)
      .interact()?;
  ```
- If selections empty: print "No tools selected." and return Ok
- Map selections to ToolKind vec
- Call `AddToolEngine::add_tools(&mut config, &selected_tools, &project_root)?`
- Run partial sync and execute (same as non-interactive)
- Print summary: for each tool, print tool name + actions count, using colored output

**Summary output format (both modes):**
```
Added: Windsurf, Codex
Synced 3 file(s) for newly added tools.
```

Use `colored` for green/yellow formatting consistent with other commands.
