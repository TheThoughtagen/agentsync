Create `crates/aisync-core/src/add_tool.rs` with:

```rust
pub struct AddToolEngine;
```

**`discover_unconfigured(config, project_root)`:**
- Call `DetectionEngine::scan(project_root)?`
- Filter results: keep only those where `config.tools.get_tool(d.tool.as_str()).is_none()`
- CRITICAL: Use `get_tool().is_none()`, NOT `!is_enabled()`. Unconfigured-is-enabled semantics mean `is_enabled()` returns true for absent tools.
- Return `Vec<DetectionResult>`

**`add_tools(config, tools, project_root)`:**
- For each tool in `tools`:
  - Get adapter via `AnyAdapter::for_tool(tool)`
  - Get `default_sync_strategy()` from adapter
  - Create `ToolConfig { enabled: true, sync_strategy }` where `sync_strategy` is `Some(strategy)` only if strategy != `SyncStrategy::Symlink` (since Symlink is the global default, omitting it keeps TOML clean)
  - Call `config.tools.set_tool(tool.as_str().to_string(), tool_config)`
- After loop, serialize config with `config.to_string_pretty()` and write to `project_root.join("aisync.toml")`
- Error mapping: serialization errors → `InitError::ImportFailed`, IO errors → `InitError::ScaffoldFailed`

Add `AddToolError` or reuse `InitError` for error mapping. Reusing `InitError` is simpler and follows precedent (init also writes aisync.toml).

Update `crates/aisync-core/src/lib.rs`:
- Add `pub mod add_tool;`
- Add `pub use add_tool::AddToolEngine;` to the pub use block

No changes to error.rs needed if reusing InitError.

Add `pub fn plan_for_tools()` to `SyncEngine`:

```rust
pub fn plan_for_tools(
    config: &AisyncConfig,
    project_root: &Path,
    only_tools: &[ToolKind],
) -> Result<SyncReport, AisyncError>
```

**Implementation strategy (per Research Pitfall 3):**
1. Run the FULL plan logic (same as `plan()`) for ALL enabled tools -- this ensures `deduplicate_actions()` sees every tool and correctly handles shared paths like AGENTS.md
2. After deduplication, FILTER `SyncReport.results` to retain only results where `tool` is in `only_tools`
3. Return the filtered report

This is the simplest correct approach: no new sync logic, full deduplication, just a post-filter.

Concretely:
- Extract the body of `plan()` into a private helper `fn plan_all_internal(config, project_root) -> Result<SyncReport, AisyncError>`
- Refactor `plan()` to call `plan_all_internal()`
- `plan_for_tools()` calls `plan_all_internal()` then filters `report.results.retain(|r| only_tools.contains(&r.tool))`

This avoids code duplication between plan() and plan_for_tools().
