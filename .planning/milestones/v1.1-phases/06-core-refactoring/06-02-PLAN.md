5. Since Plugin contains Box<dyn ToolAdapter> which is not Clone or Debug by default, implement Debug manually for AnyAdapter (format Plugin as "Plugin(...)"). Remove `#[derive(Debug, Clone)]` from AnyAdapter. Implement Clone manually — for Plugin variant, this is a compile error, so instead make AnyAdapter NOT implement Clone. Alternatively, since Plugin adapters don't need Clone for current usage, remove the Clone derive. Check if AnyAdapter is cloned anywhere; if so, keep Clone for built-in variants and panic for Plugin (or use Arc). **Research says built-in variants stay zero-cost.** The simplest approach: keep the derive but exclude Plugin from Clone. Since Rust doesn't support that, implement Clone manually — built-in variants clone normally, Plugin variant panics with "Plugin adapters cannot be cloned". Alternatively, wrap Plugin in Arc so it IS cloneable: `Plugin(Arc<dyn ToolAdapter>)`. Use Arc — it's the standard Rust pattern for shared ownership of trait objects.

**Decision: Use `Plugin(Arc<dyn ToolAdapter>)`.** This is Clone + Send + Sync automatically. Import `std::sync::Arc`.

macro_rules! dispatch_adapter {

            AnyAdapter::Cursor($inner) => $body,
            AnyAdapter::OpenCode($inner) => $body,
