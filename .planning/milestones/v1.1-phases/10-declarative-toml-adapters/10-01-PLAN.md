**DeclarativeAdapter struct:**

```rust
/// Builtin tool kind names that TOML adapters cannot shadow.
const BUILTIN_NAMES: &[&str] = &["claude-code", "cursor", "opencode", "windsurf", "codex"];

pub struct DeclarativeAdapter {
    def: DeclarativeAdapterDef,
    // Pre-computed for lifetime compatibility with ToolAdapter::conditional_tags() -> &[&str]
    // Leaked once at construction; TOML adapters live for program lifetime.
    cached_tags: Box<[&'static str]>,
    // Store instruction_path as owned for native_instruction_path() -> &str
    instruction_path: String,
    // Store display_name as owned for display_name() -> &str
    display_name_owned: String,
}
```

The `cached_tags` field uses `Box::leak` on each `String` tag at construction time, then stores the resulting `&'static str` slices. This is acceptable because TOML adapters are loaded once and live for the program's entire lifetime. The alternative (changing the trait signature) would be a breaking SDK change.

**DeclarativeAdapter::new() constructor:**
- Takes `DeclarativeAdapterDef`
- Returns `Result<Self, AdapterError>` -- returns `AdapterError::Other("...")` if name matches any BUILTIN_NAMES
- Leaks conditional tags into `cached_tags`
- Copies instruction_path and display_name into owned fields

**ToolAdapter implementation for DeclarativeAdapter:**
Follow these patterns from existing adapters:

- `name()` -> `ToolKind::Custom(self.def.name.clone())`
- `display_name()` -> `&self.display_name_owned`
- `native_instruction_path()` -> `&self.instruction_path`
- `conditional_tags()` -> `&self.cached_tags`
- `gitignore_entries()` -> `self.def.sync.gitignore_entries.clone()`
- `watch_paths()` -> computed from `self.def.sync.watch_paths` if non-empty, else `vec![self.native_instruction_path()]`. Use same leak pattern for `&str` lifetime.
- `default_sync_strategy()` -> `self.def.sync.strategy`

- `detect()`: Check each directory and file in detection config against project_root. If match_any, detected when ANY marker found. If !match_any, detected only when ALL markers found. Always return Confidence::Medium.

- `plan_sync()`:
  - For Generate strategy: Use template content with `{{content}}` replacement. Follow Windsurf pattern: create directory if needed, check if content unchanged (idempotent), create SyncAction::CreateFile.
  - For Symlink strategy: Follow Codex pattern: create SyncAction::CreateSymlink with source=project_root.join(instruction_path), target=".ai/instructions.md".
  - For Copy strategy: create SyncAction::CreateFile with canonical_content directly (no template).

- `read_instructions()`:
  - For Generate strategy with frontmatter_strip: read file, strip frontmatter delimited by the configured delimiter (same pattern as Windsurf adapter).
  - For Symlink/Copy: read the file directly, return content.

- `sync_status()`: Check if file exists (Missing if not), hash content and compare with canonical_hash (InSync or Drifted). For Generate with frontmatter, strip frontmatter before hashing.

- `translate_hooks()`: Return HookTranslation::Unsupported for all TOML adapters.

- `plan_memory_sync()`: Return empty vec (TOML adapters don't support memory sync in v1.1).

**Public helper function:**

```rust
pub fn load_toml_adapter(path: &Path) -> Result<DeclarativeAdapter, AdapterError> {
    let content = std::fs::read_to_string(path)
        .map_err(AdapterError::Io)?;
    let def: DeclarativeAdapterDef = toml::from_str(&content)
        .map_err(|e| AdapterError::Other(format!("failed to parse {}: {e}", path.display())))?;
    DeclarativeAdapter::new(def)
}
```
