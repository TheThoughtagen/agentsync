    - SyncEngine::execute handles CreateMemorySymlink (directory symlink) and UpdateMemoryReferences (managed section update)

  <action>
    **claude_code.rs -- Implement plan_memory_sync:**

    Override `plan_memory_sync` on `ClaudeCodeAdapter`. Use `MemoryEngine::claude_memory_path(project_root)` to get the target path. If memory_files is non-empty:
    1. Check if `~/.claude/projects/<key>/memory/` already exists
    2. If it's already a symlink pointing to `.ai/memory/` (canonicalized) -> return empty actions (idempotent)
    3. If it's a directory with content -> return `WarnUnsupportedHooks` variant (reuse pattern) or a new `SkipExistingFile` with reason "existing Claude memory found, run `aisync memory import claude` first"
    4. If it doesn't exist or is empty -> return `CreateMemorySymlink { link: claude_memory_path, target: project_root.join(".ai/memory") }`

    Note: The symlink goes FROM Claude's location TO .ai/memory/ (source of truth). This is a DIRECTORY symlink, not individual files.

    **opencode.rs -- Implement plan_memory_sync:**

    Override `plan_memory_sync`. Return `UpdateMemoryReferences` targeting `project_root.join("AGENTS.md")` with:
    - `marker_start: "<!-- aisync:memory -->"`
    - `marker_end: "<!-- /aisync:memory -->"`
    - `references`: one entry per memory file as `"- [topic_name](.ai/memory/topic.md)"` using relative paths from project root

    **cursor.rs -- Implement plan_memory_sync:**

    Override `plan_memory_sync`. Return `UpdateMemoryReferences` targeting `project_root.join(".cursor/rules/project.mdc")` with same markers and reference format as OpenCode.

    **sync.rs -- Extend plan() and execute():**

    In `SyncEngine::plan()`: After planning instruction sync, scan `.ai/memory/` for .md files using `MemoryEngine::list()`. If files exist, call `adapter.plan_memory_sync(project_root, &memory_files)` for each enabled tool and append those actions to the tool's result.

    In `SyncEngine::execute()`: Add handlers for the new action variants:
    - `CreateMemorySymlink { link, target }`: Create parent directories, then `std::os::unix::fs::symlink(target, link)` (directory symlink). On non-Unix, skip with a warning.
    - `UpdateMemoryReferences { path, references, marker_start, marker_end }`: Build reference block content, call `managed_section::update_managed_section(path, &refs, marker_start, marker_end)`.

  <action>
    **main.rs -- Add Memory subcommand:**

    Add to the `Commands` enum:
    ```rust
    /// Manage memory files in .ai/memory/
    Memory {
        #[command(subcommand)]
        action: MemoryAction,
    },
    ```

    Add `MemoryAction` enum:
    ```rust
    #[derive(Subcommand)]
    enum MemoryAction {
        /// List all memory files
        List,
        /// Add a new memory file
        Add {
            /// Topic name for the memory file
            topic: String,
        },
        /// Import memory from a tool's native storage
        Import {
            /// Tool to import from (currently only "claude")
            tool: String,
        },
        /// Export memory to all configured tools (same as sync for memory)
        Export,
    }
    ```

    Wire in main's match:
    ```rust
    Commands::Memory { action } => commands::memory::run_memory(action, cli.verbose),
    ```

    **commands/memory.rs -- Implement subcommands:**

    `run_memory(action, verbose)` dispatches to:

    - **list**: Call `MemoryEngine::list(project_root)`. Print each file's name (stem) and relative path. If empty, print "No memory files found. Use `aisync memory add <topic>` to create one."

    - **add**: Call `MemoryEngine::add(project_root, &topic)`. Print success message with file path. Use green colored output.

    - **import claude**: Call `MemoryEngine::import_claude(project_root)`. Print imported files. For conflicts, use `dialoguer::Confirm` to prompt per file: "Overwrite .ai/memory/{name} with Claude's version?". If confirmed, copy the file. If not TTY, skip conflicts with a warning (per Phase 2 pattern: interactive prompting in CLI layer, non-TTY uses defaults).

    - **export**: Load config, call `SyncEngine::plan()` (which now includes memory), then `SyncEngine::execute()`. Print results same as sync command. This is effectively `aisync sync` but focused on memory -- it runs the full sync since memory is now part of it.

    **commands/mod.rs -- Register module:**

    Add `pub mod memory;`
