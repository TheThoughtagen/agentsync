  <action>
1. test_init.rs - Integration tests for `aisync init`:
   - test_init_creates_ai_directory: Run init in a dir with CLAUDE.md, verify .ai/instructions.md created
   - test_init_imports_existing_claude_md: Create CLAUDE.md with content, run init, verify .ai/instructions.md has that content
   - test_init_idempotent: Run init twice, verify second run succeeds without error
   - Note: init in non-TTY mode uses defaults (no prompts), which is what we want for tests

2. test_sync.rs - Integration tests for `aisync sync` and `aisync status`:
   - test_sync_creates_tool_files: Setup project with all 3 tools, run sync, verify CLAUDE.md, AGENTS.md, .cursor/rules/project.mdc all exist
   - test_sync_dry_run_no_changes: Run sync --dry-run, verify no files created
   - test_sync_idempotent: Run sync twice, verify outputs identical (same content, same exit code)
   - test_status_after_sync_shows_in_sync: Run sync then status, verify output contains "In sync" or similar
   - test_status_json_output: Run status --json, verify output is valid JSON
   - test_check_exits_zero_after_sync: Run sync then check, verify exit code 0
   - test_check_exits_nonzero_on_drift: Run sync, manually modify a tool file, run check, verify exit code 1

3. test_round_trip.rs - Round-trip tests (QUAL-03):
   - test_round_trip_claude_code: Sync to ClaudeCode, read the CLAUDE.md content, compare to original .ai/instructions.md (for symlink adapter, content should be identical)
   - test_round_trip_opencode: Sync to OpenCode, read AGENTS.md, compare to original (symlink adapter)
   - test_round_trip_cursor: Sync to Cursor, read .cursor/rules/project.mdc, verify the body content (after stripping YAML frontmatter) matches original instructions
   - test_round_trip_with_conditionals: Create instructions.md with conditional sections (`<!-- aisync:claude-only -->content<!-- /aisync:claude-only -->`), sync, verify Claude has conditional content while Cursor does not

All tests should use the shared helpers from mod.rs. Use `assert_fs::prelude::*` for file assertions and `predicates::prelude::*` for output assertions.

Important: The aisync binary expects to run from a project directory. Always use `.current_dir(temp.path())` on the Command.

Important: For init tests, the binary is non-interactive in non-TTY mode (test runner is non-TTY). It will use defaults.
