    ```rust
    /// Options controlling init behavior. CLI layer populates these from user choices.
    pub struct InitOptions {
        pub force: bool,           // Re-init even if .ai/ exists
        pub import_from: Option<ImportChoice>, // Which source to import from (decided by CLI layer)
    }

    /// A discovered source of existing instructions.
    pub struct ImportSource {
        pub tool: ToolKind,
        pub content: String,
        pub source_path: PathBuf,
    }

    /// User's choice about which import source to use.
    pub enum ImportChoice {
        UseSource(ToolKind),  // Use specific tool's content
        Skip,                 // Don't import, start fresh
    }

    pub struct InitEngine;

    impl InitEngine {
        /// Check if already initialized (.ai/ directory exists).
        pub fn is_initialized(project_root: &Path) -> bool

        /// Detect tools in the project (delegates to DetectionEngine).
        pub fn detect_tools(project_root: &Path) -> Result<Vec<DetectionResult>, AisyncError>

        /// Find existing instruction sources from detected tools.
        /// Calls read_instructions() on each detected adapter.
        pub fn find_import_sources(project_root: &Path, detected: &[DetectionResult]) -> Vec<ImportSource>

        /// Scaffold the .ai/ directory structure.
        /// Creates: .ai/instructions.md, .ai/memory/, .ai/hooks/, .ai/commands/, aisync.toml
        /// If import_content is Some, writes it to .ai/instructions.md.
        /// Otherwise creates empty .ai/instructions.md.
        /// Writes aisync.toml with detected tools enabled (or empty tools if none detected).
        pub fn scaffold(
            project_root: &Path,
            detected_tools: &[DetectionResult],
            import_content: Option<&str>,
            options: &InitOptions,
        ) -> Result<(), AisyncError>
    }
    ```

    The key insight: **Keep all interactive prompting out of aisync-core.** The core library discovers data (detect, find sources) and executes actions (scaffold). The CLI layer (Task 2) handles all dialoguer prompts and passes decisions back.

    **aisync.toml generation:** Build an AisyncConfig struct from detected tools:
    - schema_version = 1
    - defaults.sync_strategy = Symlink
    - For each detected tool: set tools.{tool} = ToolConfig { enabled: true, sync_strategy: None }
    - Cursor always gets sync_strategy: Some(Generate) since it cannot use symlinks
    - Serialize with `config.to_string_pretty()` and write to project_root/aisync.toml

    Add `pub mod init;` to lib.rs and re-export InitEngine, InitOptions, ImportSource, ImportChoice.

    **crates/aisync/src/commands/init.rs:**

    Implement `pub fn run_init(verbose: bool) -> Result<(), Box<dyn std::error::Error>>`:

    1. **Check existing init:**
       - If `InitEngine::is_initialized(".")`, use `dialoguer::Confirm` to ask:
         "`.ai/` directory already exists. Re-initialize? This will overwrite aisync.toml and re-import instructions."
       - If user declines, exit with message "Aborted."
       - If user confirms, set `force = true`

    2. **Detect tools:**
       - Call `InitEngine::detect_tools(".")`
       - Print detected tools with colored output: "Found: Claude Code, Cursor" (green)
       - If no tools detected, print: "No AI tools detected. Proceeding with empty configuration." (yellow)
       - Use `dialoguer::Confirm`: "Proceed with these tools?" (default: true)
       - If user declines, exit with "Aborted."

    3. **Import existing configs:**
       - Call `InitEngine::find_import_sources(".", &detected)`
       - If 0 sources: skip import, create empty instructions
       - If 1 source: Use `dialoguer::Confirm`: "Import instructions from {tool} ({path})?"
         - If yes, use that content
       - If 2+ sources (INST-06 conflict resolution):
         - Print each source with a preview (first 5 lines)
         - Use `dialoguer::Select` to let user pick which source to import
         - Options: each tool name + "Start fresh (empty)"

    4. **Scaffold:**
       - Call `InitEngine::scaffold(".", &detected, import_content, &options)`
       - Print summary of what was created (colored, green checkmarks):
         - "Created .ai/instructions.md"
         - "Created .ai/memory/"
         - "Created aisync.toml (3 tools configured)"

    5. **Non-TTY handling:** Check `std::io::stdin().is_terminal()`. If non-TTY:
       - Skip all prompts
       - Default to: proceed with detected tools, import from first source (or none)
       - Print warning: "Non-interactive mode: using defaults"

    Note: This task creates the commands directory structure. main.rs will be updated in Plan 04 to wire clap subcommands to these command handlers.
