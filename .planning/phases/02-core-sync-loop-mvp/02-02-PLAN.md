    Create crates/aisync-core/src/sync.rs with `SyncEngine`:

    ```rust
    pub struct SyncEngine;

    impl SyncEngine {
        /// Plan sync actions for all enabled tools. Does not modify filesystem.
        /// Used by both `aisync sync` and `aisync sync --dry-run`.
        pub fn plan(config: &AisyncConfig, project_root: &Path) -> Result<SyncReport, AisyncError>

        /// Execute a planned sync. Mutates filesystem.
        /// `interactive` flag controls whether SkipExistingFile actions prompt the user
        /// (the actual prompting is done in the CLI layer — this just filters or includes those actions).
        /// Non-interactive mode: skip SkipExistingFile actions silently.
        /// After executing tool syncs, update .gitignore with aisync-managed section (INST-07).
        pub fn execute(actions: &SyncReport, project_root: &Path) -> Result<SyncReport, AisyncError>

        /// Get sync status for all enabled tools.
        pub fn status(config: &AisyncConfig, project_root: &Path) -> Result<StatusReport, AisyncError>
    }
    ```

    **Plan logic:**
    1. Read canonical content from `project_root/.ai/instructions.md` — error if missing (CanonicalMissing)
    2. For each enabled tool in config: call `adapter.plan_sync(project_root, canonical_content, strategy)`
    3. Collect results into SyncReport — continue on error per tool (partial sync failure pattern)

    **Execute logic:**
    1. For each ToolSyncResult in the report, execute each SyncAction:
       - CreateSymlink: `std::os::unix::fs::symlink(target, link)` (unix) or `std::fs::copy` (windows)
       - RemoveAndRelink: `fs::remove_file(link)` then create symlink
       - GenerateMdc: `fs::create_dir_all(parent)` then `fs::write(output, content)`
       - CreateDirectory: `fs::create_dir_all(path)`
       - CreateFile: `fs::write(path, content)`
       - UpdateGitignore: call `gitignore::update_managed_section`
       - SkipExistingFile: record as skipped (no filesystem change)
    2. After tool syncs, compute gitignore entries based on what was synced:
       - Symlinked files: `CLAUDE.md`, `AGENTS.md`
       - Generated files: `.cursor/rules/project.mdc`
    3. Call `gitignore::update_managed_section` with computed entries

    **Status logic:**
    1. Read canonical content, compute SHA-256 hash
    2. For each enabled tool: call `adapter.sync_status(project_root, canonical_hash)`
    3. Collect into StatusReport

    Add `pub mod sync;` to lib.rs and re-export `SyncEngine`.
